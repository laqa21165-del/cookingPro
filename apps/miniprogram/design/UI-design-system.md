# 「文字点单」小程序 UI 设计方案

> UI Designer · 基于现有排版与品牌调性的全新界面设计
> 范围：设计系统规范 + 首页（Home）效果预览
> 状态：提案稿，不含代码改动

---

## 1. 设计目标

在不破坏现有「暖奶油 + 陶土红 + 棕墨」品牌资产的前提下，解决当前首页的 3 个核心问题：

1. **视觉层次不够清晰** —— 英雄区、分享卡、开始点菜、最近厨师四块信息权重接近，用户难以一眼抓住重点。
2. **「首页」字标冗余** —— 顶部 eyebrow 写「首页」对使用者没有信息增量，应换成有温度的问候与时间语境。
3. **缺乏品牌记忆点** —— 小狗吉祥物目前被塞进一张小卡，应成为贯穿全站的「陪伴伙伴」，强化「被认真阅读、好好回应」的情感价值。

设计主张：**有机自然 · 柔和治愈（Soft Organic & Healing）**——像一杯刚煮好的燕麦拿铁，圆润、温润、有人情味，但结构清晰、操作不费力。

## 2. 设计上下文

| 维度 | 说明 |
|------|------|
| **目标用户** | 与「专属厨师」建立长期绑定关系的食客。点单是高频、重复的双向照顾行为。 |
| **使用场景** | 打开小程序 → 看到当日问候 → 一键开始点菜 / 选择某位厨师继续点单 / 管理绑定关系。 |
| **品牌个性** | 温暖、亲密、被照顾、有人情味。「用文字付款」，每一次点单都被认真阅读。 |
| **差异化记忆点** | 会说话的小狗吉祥物「汪」，作为用户与厨师之间的心意信使。 |

## 3. 设计令牌（Design Tokens）

### 3.1 色彩系统（保留品牌色，提炼为可扩展色阶）

| Token | 值 | 用途 | 对比度（WCAG AA） |
|-------|-----|------|-----------|
| `--cream-50` | `#FFF7F2` | 页面背景底色 | — |
| `--cream-100` | `#FDEFE7` | 卡片辅助底、区块分隔 | — |
| `--surface` | `#FFFDFB` | 主卡片表面 | 文本对比 ≥ 4.5:1 |
| `--ink` | `#3B251F` | 主标题 / 正文 | 对 cream 7.1:1 ✅ |
| `--ink-soft` | `#5E463D` | 次级标题 | 5.3:1 ✅ |
| `--muted` | `#8E6F67` | 辅助说明文字 | 对 surface 4.6:1 ✅ |
| `--primary` | `#C2674E` | 主按钮 / 关键强调 | 白字 4.6:1 ✅ |
| `--primary-600` | `#A8533D` | 主按钮按压 / 深强调 | 白字 6.2:1 ✅ |
| `--primary-100` | `#F6E0D7` | 头像底、浅强调面 | — |
| `--line` | `rgba(155,103,84,.14)` | 描边、分隔线 | — |
| `--sage` | `#5F8B5B` | 成功 / 绑定确认 | 白字 4.5:1 ✅ |
| `--sage-bg` | `#EBF4E7` | 成功提示背景 | — |

> 注：对比度基准沿用现有主色 `#cb6d5a` 微调为 `#C2674E`，在保持品牌识别的同时提升白字可读性至 AA。

### 3.2 字体系统

- **主字体**：`PingFang SC`（与现有一致，iOS 原生最佳中文字体）
- **字阶**（基于 4px 基准的流式比例）：

| 角色 | 字号 | 字重 | 行高 | 示例 |
|------|------|------|------|------|
| 展示标题 | 30px | 800 | 1.2 | 今天想吃什么？ |
| 区块标题 | 21px | 700 | 1.24 | 最近可点单的厨师 |
| 卡片标题 | 18px | 700 | 1.3 | 厨师名 |
| 正文 | 15px | 400 | 1.6 | 辅助说明 |
| 小字 / eyebrow | 12px | 600 | 1.4 | 晚上好 · 把心意交给文字 |

### 3.3 间距与圆角

- **基准单位**：4px；常用节奏 `8 / 12 / 16 / 20 / 28 / 40`
- **圆角**：卡片 `26px`、按钮 `18px`、头像 `16px`、chip `999px`
- **阴影**：单层柔光 `0 14px 30px rgba(124,86,68,.10)`，避免多层重影

### 3.4 动效原则

- 状态变化用 `ease-out` 指数缓动（≈220ms）
- 仅过渡 `transform` / `opacity` / `box-shadow`，不动 layout 属性
- 尊重 `prefers-reduced-motion`，关闭位移与缩放

## 4. 组件库（首页涉及）

| 组件 | 规范要点 |
|------|----------|
| **顶栏问候** | 左：时段问候 + 一句话心意；右：圆形头像入口。无边框、融入背景。 |
| **英雄卡片** | 暖色微渐变 + 小狗探头；大展示标题 + 一行说明。 |
| **主行动按钮（Primary CTA）** | 整行陶土渐变，「开始点菜」+ 箭头；按压下沉 1px。 |
| **快捷操作 chip** | 浅面胶囊，左图标右文字：「邀请顾客绑定」「绑定专属厨师」。 |
| **厨师卡片** | 横向滑动；方形头像（姓氏首字）+ 名称 + 「继续点单」。 |
| **空状态** | 小狗居中 + 一句引导文案 + 主按钮，替代冷冰冰的占位。 |
| **提示条** | 成功用 sage 浅底；错误用陶土浅底，统一圆角与内距。 |
| **底部标签栏** | 4 项 + 图标（首页/订单/菜单/我的），选中态陶土色 + 上移 2px。 |

> 现有 tabBar 仅有文字、无图标。建议补齐线性图标资产（预览中已示意），可显著降低首屏认知负载。

## 5. 首页信息架构（Before → After）

**Before（现有堆叠）**
```
英雄卡(含冗余"首页"字标 + 小狗卡 + 两个按钮)
├─ 绑定确认提示（条件）
├─ 模拟分享卡（条件，含 token 框）
├─ 开始点菜大卡
└─ 最近可点单的厨师（纵向网格）
```

**After（重构层次）**
```
顶栏问候（时段 + 心意，替代"首页"字标）
├─ 英雄卡（问候标题 + 小狗探头 + 一行价值说明）
├─ 主 CTA：开始点菜（唯一最高权重）
├─ 快捷关系操作：邀请顾客绑定 / 绑定专属厨师（次级）
├─ 最近可点单的厨师（横向滑动，空态用小狗引导）
└─ 底部标签栏（含图标）
```

**关键改动**
1. 删除英雄区的「首页」字标，改为有温度的时段问候，降低认知噪音。
2. 「开始点菜」从并列卡片升级为页面唯一主 CTA，符合用户最高频意图。
3. 邀请 / 绑定从主操作降为次级 chip，避免与 CTA 争夺注意力。
4. 厨师列表由纵向网格改为横向滑动，首屏露出更多、操作更轻。
5. 小狗吉祥物从「被塞进小卡」升级为英雄区探头 + 空状态陪伴，强化品牌记忆。

## 6. 可访问性（WCAG AA）

- 正文 / 辅助文字对比度均 ≥ 4.5:1（见 3.1 表）
- 所有可点元素 ≥ 44px 触控区
- 图标按钮配 `aria-label`，标签栏图标 + 文字双标识
- 支持系统字号放大至 200% 不破版
- 尊重「减少动态效果」系统偏好

## 7. 暗色模式（简述，后续页面扩展）

以暖棕为底（`#2A1C17`），表面提亮为 `#3A2A22`，主色提亮为 `#E08A6E`，
狗与强调元素保持暖调发光，避免冷色霓虹。预览文件底部提供「切换暗色」按钮可即时体验。
